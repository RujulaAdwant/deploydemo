Claude → VS Code → GitHub → Vercel

A step-by-step guide to copying code from Claude, versioning it with Git, and deploying it live on the web.

Part 1 — Get your code into VS Code

1
Copy the artifact from Claude
In Claude's artifact panel, click the copy icon in the top-right corner to copy all the code to your clipboard.
2
Open VS Code and create a file
Go to File → Open Folder and open (or create) a project folder. In the Explorer sidebar, right-click and choose New File. Name it (e.g. index.html) and paste your code in.

Part 2 — Set up Git & GitHub

3
Initialize Git in the terminal
Open the terminal in VS Code (Terminal → New Terminal). Run the command below. This turns your folder into a Git repository that can track changes.
git init
⚠️ On older Macs, git init creates a branch called master instead of main. To avoid headaches, run this once to fix it globally: git config --global init.defaultBranch main
4
Create a repo on GitHub and connect it
Go to github.com, click + → New repository. Name it, leave it empty (no README), and click Create repository. Then run this in the terminal:
git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO.git
5
Commit and push your code
Run these three commands. add selects files, commit saves a snapshot, push uploads it to GitHub.
git add .
git commit -m "Add code from Claude"
git push -u origin main

Part 3 — Deploy on Vercel

6
Import your repo on Vercel
Go to vercel.com and sign up (you can log in with GitHub). Click Add New Project, then import your GitHub repo. Vercel auto-detects your project type — just click Deploy.
7
Your site is live — and auto-updates forever
Vercel gives you a public URL like my-project.vercel.app. From now on, every time you push code to GitHub, Vercel automatically redeploys. No manual steps needed.
Tips: The free Vercel "Hobby" tier covers most personal projects. Connect a custom domain under Settings → Domains. If anything goes wrong, copy the error message and paste it into Claude — see the Ask Claude tab for helpful prompts.

Troubleshooting

Click any issue to see the fix. When in doubt, paste the error into Claude.

Git issues

Branch is called master instead of main Git▼

This happens on older versions of Git where master is still the default branch name. GitHub expects main, so the push fails.

Quick fix — just push to master instead:

git push -u origin master

Permanent fix — rename your branch and push:

git branch -m master main
git push -u origin main

Fix it for all future repos — run this once:

git config --global init.defaultBranch main
error: src refspec main does not match any Git▼

This usually means you haven't made your first commit yet, or your local branch is named differently than what you're pushing to.

Check what your branch is actually called:

git branch

Then push using that name (e.g. master):

git push -u origin master

Or make sure you've committed something first — git push has nothing to send if you haven't run git commit.

The current branch has no upstream branch Git▼

Git doesn't know which remote branch to push to. Use -u to set it the first time:

git push -u origin main

After this, plain git push will work for all future pushes.

Permission denied (publickey) Git▼

Git can't authenticate with GitHub. The easiest fix is to use HTTPS instead of SSH for your remote URL.

Update your remote to use HTTPS:

git remote set-url origin https://github.com/YOUR_USERNAME/YOUR_REPO.git

GitHub will ask for your username and a personal access token (not your password) — generate one at github.com/settings/tokens.

Nothing to commit, working tree clean Git▼

Git sees no changes since your last commit. This isn't an error — it just means you haven't edited any files yet, or you forgot to save them in VS Code.

Save your files (Cmd+S / Ctrl+S), then try git add . and git commit again.

GitHub issues

Repository not found GitHub▼

The remote URL is wrong, or the repo doesn't exist yet. Double-check the URL:

git remote -v

If it's wrong, update it:

git remote set-url origin https://github.com/YOUR_USERNAME/YOUR_REPO.git
Failed to push — remote contains work you don't have locally GitHub▼

GitHub has commits your local repo doesn't. This can happen if you created a README on GitHub. Pull first, then push:

git pull origin main --rebase
git push -u origin main

Vercel issues

Build failed on Vercel Vercel▼

Open the failed deployment in Vercel and click View Build Logs. Copy the red error lines and paste them into Claude — it can usually diagnose and fix the problem quickly.

Site deployed but shows a blank page or 404 Vercel▼

Vercel can't find your entry file. Make sure your main file is named index.html (or index.js for Node projects) and is in the root of your repo, not inside a subfolder.

In Vercel's project settings, check that the Root Directory is set correctly.
